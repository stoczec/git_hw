1. git checkout имя файла
2. git reset HEAD имя файла
3. git reset --soft
4. git checkout -b название новой ветки
5. fast-forward merge происходит при слиянии веток, например ветка "1" в ветку "2". При условии, что изменения в ветке "1" были после коммита в ветке "2".
   non fast-forfard merge происходит при слиянии веток. При условии, что коммит в ветке "1" был после изменения в ветке "2".
6. Когда есть изменения в одном и том же файле + в одной и той же строке. Разрешить конфликт можно в ручную, нужно выбрать правильное изменение,
   и потом его закомитить.
7. Можно. git branch -d имя ветки
